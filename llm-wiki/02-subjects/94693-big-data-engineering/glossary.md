# 94693 Big Data Engineering - Glossary

| Term | Evidence Count | Working Definition |
|---|---:|---|
| batch processing | 0 | Processing data in bounded groups at scheduled intervals, often at scale with predictable throughput. |
| data lake | 0 | A storage zone for raw or semi-structured data before full curation and schema enforcement. |
| data warehouse | 0 | A structured repository designed for consistent reporting, BI, and query performance. |
| ETL | 0 | Extract-transform-load workflow for moving data into reliable analytical systems. |
| orchestration | 0 | Coordination of dependent data jobs with scheduling, retries, and dependency tracking. |
| pipeline | 0 | End-to-end data flow from ingestion through transformation to storage and consumption. |
| partitioning | 0 | Splitting data into groups to improve query and processing efficiency. |
| scalability | 0 | The ability to handle larger datasets/throughput with additional resources. |
