# 36118 Applied Natural Language Processing - Glossary

| Term | Evidence Count | Working Definition |
|---|---:|---|
| tokenisation | 123 | Splitting raw text into units such as words, subwords, or tokens that downstream NLP models can process. |
| large language models | 111 | Large pretrained language models used for generation, classification, extraction, summarisation, and reasoning-style workflows. |
| word embeddings | 93 | Dense vector representations that capture semantic or contextual relationships between words or text units. |
| transformers | 82 | Attention-based neural architectures that power models such as BERT and many modern NLP systems. |
| text preprocessing | 70 | Cleaning and transforming text before analysis, including normalisation, stopword handling, stemming, lemmatisation, and filtering. |
| clustering | 67 | Grouping documents or text units by representation similarity without using labelled classes. |
| deep learning for text | 61 | Neural models for sequence or text tasks, including RNNs, LSTMs, CNNs, and transformer-based models. |
| text classification | 50 | Assigning one or more labels to text, usually with supervised learning and task-specific metrics. |
| topic modelling | 47 | Unsupervised methods for discovering recurring themes or latent topics in a document collection. |
| sentiment analysis | 47 | Estimating opinion, polarity, or affect from text. |
| bag of words | 41 | A sparse representation that counts words or weighted word occurrences without modelling word order. |
| responsible ai | 25 | Considering bias, privacy, consent, fairness, transparency, and downstream impact in NLP systems. |
| summarisation | 21 | Producing a shorter representation of text while preserving the important content for a task. |
| evaluation | 18 | Measuring output quality with task-appropriate metrics, validation data, qualitative review, and error analysis. |
