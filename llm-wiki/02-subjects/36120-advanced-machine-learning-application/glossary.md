# 36120 Advanced Machine Learning Application - Glossary

| Term | Evidence Count | Working Definition |
|---|---:|---|
| experiment tracking | 562 | Systematic logging of setup, code, and outputs for review and audit. |
| overfitting | 504 | A pattern where a model performs well on training data but poorly generalizes. |
| reproducibility | 321 | Ability to re-run an experiment and obtain consistent outcomes. |
| model evaluation | 318 | Measuring model quality using metrics aligned to a defined objective and failure cost. |
| data leakage | 234 | Including information unavailable at prediction time, producing inflated validation scores. |
| feature engineering | 167 | Transforming raw inputs into forms that improve model signal and stability. |
| class imbalance | 131 | When target classes are not equally represented and metrics can mislead. |
| ensemble | 96 | Combining multiple models to improve robustness and reduce variance. |
| hyperparameter tuning | 76 | Systematic selection of algorithm settings that are not learned directly from gradients. |
| evaluation protocol | 53 | Consistent strategy for splits, folds, and metric reporting. |
| time-series | 46 | Data indexed by time where ordering and dependencies constrain validation strategy. |
| probability calibration | 5 | Adjusting predicted probabilities to better reflect true outcome frequencies. |
