# 36106 Machine Learning Algorithms and Applications - Glossary

| Term | Evidence Count | Working Definition |
|---|---:|---|
| feature engineering | 146 | Creating, transforming, encoding, or selecting variables to improve model learning. |
| regression | 119 | A supervised learning task where the model predicts a continuous target. |
| supervised learning | 112 | Learning a mapping from input features to known target labels or values. |
| decision tree | 95 | A model that predicts by recursively splitting feature space into decision regions. |
| model evaluation | 70 | Measuring model performance with metrics aligned to the problem and consequences. |
| classification | 66 | A supervised learning task where the model predicts a discrete class or class probability. |
| anomaly detection | 63 | Identifying observations that are unusual relative to expected patterns. |
| dimensionality reduction | 47 | Transforming data into fewer variables while retaining important structure. |
| clustering | 37 | Unsupervised grouping of examples based on similarity or distance structure. |
| ensemble learning | 15 | Combining multiple models to improve predictive performance or robustness. |
| hyperparameter tuning | 14 | Choosing model settings that are not directly learned from the training algorithm. |
| model interpretation | 11 | Explaining what a model learned, which features matter, and where predictions may fail. |
| train-test split | 9 | A separation of data used to estimate performance on examples not used for fitting. |
| cross-validation | 3 | A validation strategy that repeatedly trains and evaluates models across different folds. |
| data leakage | 2 | Using information during training that would not be available at prediction time. |
